\section{Introduction}

本课程作业围绕空间智能中的两个互补问题展开：其一是从多种来源构建、对齐并融合三维资产，其二是训练机器人模仿学习策略并检验其跨环境泛化能力。第一条主线关注三维内容生成与真实场景融合。我们分别使用真实多视角图像、文本提示和单张图像作为输入，得到物体级三维表示，再将这些资产放入真实背景场景中完成统一相机轨迹下的渲染输出。第二条主线关注 CALVIN 机器人操作任务中的 ACT 策略训练。我们基于课程提供的官方 \texttt{xiaoma26/calvin-lerobot} 数据划分，在环境 A 上训练 A-only 模型，并与环境 A/B/C 联合训练模型进行对比，最终在未见环境 D 上评估动作误差与可行的 rollout 表现。

这两条主线共享同一个核心要求：三维表示、坐标系、数据接口和实验流程必须足够可复现。对于三维生成部分，物体资产可能来自 3D Gaussian Splatting、text-to-3D 或 image-to-3D，不同表示之间的尺度、朝向、纹理质量和渲染管线存在明显差异；最终视频质量取决于这些中间表示能否在同一个世界坐标系下稳定组合。对于机器人学习部分，策略效果受到数据 split、动作空间、视觉输入解析、训练步数预算和评测协议的共同影响；若 A-only 与 ABC 训练使用不同的有效优化预算，对比结论会混入算力差异。因此，本报告将工程可追踪性作为实验设计的一部分，记录数据转换、训练配置、日志指标、checkpoint 指针和评测路线。

我们的预期交付物包括：题目一的三类物体资产、真实背景融合视频、三种三维生成路径的质量与耗时对比；题目二的 LeRobot ACT 训练脚本、A-only 与 ABC 固定步数训练结果、环境 D 上的 zero-shot 对比指标，以及动作分块鲁棒性分析。当前章节先给出任务动机与相关工作，最终实验数值将在 Experiments 和 Discussion 中补充。

\section{Related Work}

三维重建与新视角合成是空间智能的基础环节。COLMAP 通过结构光束法和多视角几何估计相机位姿与稀疏点云，是许多神经渲染方法的前置步骤\cite{schonberger2016sfm,schonberger2016mvs}。3D Gaussian Splatting 将场景表示为可微的各向异性高斯集合，在保持高质量新视角合成的同时显著提升实时渲染效率\cite{kerbl2023gaussian}。在本作业中，真实物体和背景场景均依赖多视角重建与相机轨迹估计，因此注册率、点云清理、尺度对齐和渲染轨迹直接影响最终融合视频。

文本或单图驱动的三维生成扩展了资产来源。DreamFusion 使用预训练文本到图像扩散模型提供 score distillation signal，从文本提示优化三维表示\cite{poole2022dreamfusion}；Zero123 学习从单张图像合成新视角，为单图到三维优化提供了视角先验\cite{liu2023zero123}；Stable Zero123 和 threestudio 将这些思路整理为可复用的开源管线，便于在课程项目中完成 prompt 设计、SDS 优化、mesh 导出和质量检查\cite{stability2023zero123,threestudio2023}。这些方法输入成本低，但几何一致性、Janus 伪影、纹理漂移和尺度控制仍需要人工审计。

机器人模仿学习部分与长时程语言条件操作基准相关。CALVIN 提供了跨环境的语言条件桌面操作任务，用于评估策略在不同视觉布局和物体配置下的泛化能力\cite{mees2022calvin}。ACT 使用 Transformer 预测一段动作 chunk，通过分块动作降低闭环控制频率并提升长时程操作的稳定性\cite{chi2023act}。LeRobot 提供统一的数据集格式、策略配置和训练入口，降低了从离线数据到策略 checkpoint 的工程成本\cite{lerobot2024}。本作业沿用 ACT 的动作分块思想，并重点检查 A-only 与 ABC 数据覆盖对环境 D zero-shot 表现的影响。

\subsection{3D Fusion Assets}

题目一的背景场景选择 Mip-NeRF 360 counter。该场景提供真实厨房台面图像和 COLMAP 相机参数，适合作为多物体融合的真实背景。项目先对 counter 进行 3D Gaussian Splatting 训练，并使用同一相机轨迹渲染新视角背景；后续融合阶段以 counter 的 COLMAP/3DGS 坐标系作为世界坐标系基准，保证插入物体在多视角视频中保持稳定的空间关系。

真实物体 \texttt{object\_A} 由手机环绕采集得到。受家庭拍摄条件限制，采集地点为陶瓷桌面，最终选用一段连续环绕视频作为重建输入，以获得更稳定的 COLMAP 注册。该视频完成抽帧、SfM 注册和 3DGS 训练后，COLMAP 注册率达到 100\%，30k step 训练的 test PSNR 记录为 31.33。报告中将把 object\_A 作为真实多视角重建路径的代表，并透明说明其点云散点和桌面采集条件对外观质量的影响。

生成式物体用于补充真实采集之外的资产来源。物体 B 采用 threestudio / DreamFusion 类 text-to-3D 路线，当前选择 hamburger prompt，已完成 10000 step 训练并导出 textured mesh。物体 C 采用 Stable Zero123 image-to-3D 路线，输入为主体居中的单图 RGBA，导出形式为 OBJ/MTL 和纹理文件。B/C 两个 mesh 会在 Blender 中检查材质、尺度和朝向，再与 counter 背景、object\_A 高斯表示共同组成最终融合视频。

\section{Methods}

\subsection{Topic 1: Multi-Source 3D Asset Fusion}

题目一采用多源三维资产融合流程。我们将资产来源分为真实多视角重建、文本驱动生成、单图驱动生成和真实背景场景四类，并在最终渲染阶段把它们对齐到同一相机轨迹下。整体设计的重点是保留每种方法最稳定的中间表示：真实物体和背景使用 3D Gaussian Splatting 表示，文本和单图生成物体导出为 textured mesh，再通过 Blender 场景或 pass-level 合成完成视频输出。

真实物体 A 使用手机环绕视频作为输入。视频先通过 \texttt{ffmpeg} 抽帧并过滤模糊帧，随后使用 COLMAP 进行 Structure-from-Motion，恢复相机内外参、稀疏点云和图像注册关系。注册结果经 3DGS 的 \texttt{convert.py} 转换为训练数据后，使用 3D Gaussian Splatting 优化物体级高斯集合。该路径的优势是几何和纹理直接来自真实观测，质量主要受拍摄轨迹、特征匹配质量、点云清理和训练迭代数影响。Week 1 中物体 A 已完成单圈视频重建，COLMAP 注册率达到 100\%，并完成 7k/30k 两档 3DGS 训练。

文本生成物体 B 使用 threestudio 的 DreamFusion 类流程。给定文本提示后，系统以随机初始化的三维表示作为优化变量，通过预训练 text-to-image 扩散模型提供 score distillation signal，使多视角渲染结果逐步符合文本语义。训练完成后通过 threestudio 的导出接口生成 OBJ/MTL 和纹理文件，供 Blender 读取。该路径的输入成本最低，适合快速产生语义明确的资产；风险集中在 Janus 伪影、局部几何漂浮和材质不稳定。当前项目选择 hamburger 作为物体 B，原因是该资产已完成 10000 step 训练、mesh 导出和 Blender 打开验证，并且与 counter 厨房台面背景语义一致。

单图生成物体 C 使用 Stable Zero123 路线。输入是一张主体居中、背景干净的正面 RGBA 图像。Stable Zero123 先提供图像条件的新视角先验，再结合 SDS 优化三维表示，使渲染结果在不同视角下保持与输入物体一致。执行时需要显式记录输入视角假设，例如 elevation 与 azimuth；若首轮出现双面结构、漂浮块或轮廓破碎，则优先调整 seed、prompt 约束和 Perp-Neg 设置。最终交付形式同样是 textured mesh，便于与物体 B 共用 Blender 放置、缩放和材质检查流程。

背景场景选择 Mip-NeRF 360 的 counter。背景首先按 3DGS 流程进行 COLMAP/3DGS 训练，得到可沿真实相机轨迹渲染的新视角场景。融合阶段以 counter 的 COLMAP/3DGS 坐标系作为世界坐标系基准：物体 B 和 C 的 mesh 在 Blender 中手动或脚本化调整尺度、朝向和位置，使它们稳定落在台面区域；物体 A 维持高斯表示时，可使用同轨迹的双 pass 渲染和 alpha 合成加入最终视频。若 mesh 与 Gaussian 的单一渲染管线难以统一，报告中将明确采用 pass-level 合成策略，并保留每个 pass 的相机参数、帧序列和 ffmpeg 合成命令，保证最终视频可复现。

\subsection{Topic 2: ACT/CALVIN Cross-Environment Generalization}

题目二采用 Action Chunking Transformer (ACT) 作为模仿学习策略。ACT 以当前视觉观测和机器人状态为输入，通过 Transformer encoder-decoder 预测一段未来动作序列。项目配置使用 \texttt{chunk\_size=100} 与 \texttt{n\_action\_steps=100}，动作维度为 7，前 6 维是相对末端位姿增量，最后一维是夹爪命令。训练时模型对完整动作块计算 L1 损失，并用 \texttt{action\_is\_pad} 屏蔽 episode 末尾越界的 future action。这个动作分块设计降低了策略查询频率，也让后续分析可以按 chunk 内时间位置统计误差，观察跨环境视觉分布偏移对短期和长期动作预测的影响。

训练入口由 \texttt{topic2\_act/scripts/run\_act\_train.py} 封装。官方 \texttt{xiaoma26/calvin-lerobot} 数据以 v2.1 schema 发布，原始字段为 \texttt{state}、\texttt{actions}、\texttt{image} 和 \texttt{wrist\_image}；LeRobot ACT 训练需要 canonical key，因此脚本将字段重命名为 \texttt{observation.state}、\texttt{action}、\texttt{observation.images.image} 和 \texttt{observation.images.wrist\_image}，再转换到 LeRobot v3 数据布局。A-only 模型只使用环境 A，ABC 模型使用环境 A/B/C 的合并 canonical dataset。两组正式训练共享网络结构、batch size、学习率、weight decay、随机种子、DataLoader 配置和 150000 gradient steps，只改变训练数据范围，从而保证环境覆盖差异是主要对比变量。

评测使用两层指标。主指标是 splitD 离线 open-loop Action L1：评测脚本直接读取环境 D 的 raw parquet，按 ACT 配置构造 100 步 future action chunk，加载 checkpoint 保存的 preprocessor 和 policy，在 normalized action 空间计算有效动作元素上的平均 L1，并额外保存按动作维度、chunk 位置和原始动作单位反归一化后的误差统计。该指标不依赖仿真图形栈，能够稳定给出 A-only 和 ABC 在未见环境 D 上的 zero-shot 动作预测误差。辅指标是 CALVIN closed-loop rollout success rate 或 smoke success；它依赖 PyBullet/CALVIN 渲染路径，因此只在 EGL 或 fallback 路线可运行时记录。

CALVIN 官方评测环境固定在 Python 3.8，而 LeRobot 0.4.0、ACT checkpoint 和训练侧依赖运行在 Python 3.12。为避免在 CALVIN 环境中复刻训练栈，项目采用双环境桥接架构：Python 3.8 的 \texttt{LeRobotACTWrapper} 只保留 CALVIN 的 \texttt{reset()} 和 \texttt{step(obs, goal)} 接口，真实图像缩放、状态整理、normalization、ACT 推理和 action chunk 队列都由 Python 3.12 worker 完成。每次 CALVIN 子任务开始时 wrapper 调用 worker reset，清空 action 队列，保证不同 rollout 之间没有历史动作残留。

\begin{verbatim}
CALVIN py3.8 env
  reset()/step(obs, goal)
        |
        |  4-byte length + pickle protocol 4 frame
        v
LeRobot py3.12 worker
  raw CALVIN obs -> canonical ACT batch
  checkpoint preprocessor -> ACTPolicy.select_action
  list[float] action[7] -> CALVIN env.step(action)
\end{verbatim}

桥接协议只在 stdin/stdout 上传输带长度前缀的 pickle frame，worker 日志写入 stderr 或独立日志文件。worker 返回的动作 payload 使用纯 Python \texttt{list[float]}，CALVIN 侧再转换为 \texttt{np.float32[7]}，这样可以避开 Python 3.12/NumPy 2 到 Python 3.8/NumPy 1 之间的 ndarray pickle 兼容问题。EGL 路线按 Day 4 决策一次性处理：优先尝试 strict NVIDIA EGL；若仍受系统 GLVND/NVIDIA userspace 限制，则采用 Day 6 已验证的 \texttt{direct-cameras} 路线作为 closed-loop 辅助证据，并在报告中明确该 fallback 的使用范围。Day 4 实测中，系统包修复后 strict EGL 仍停留在 Mesa llvmpipe，因此后续 closed-loop smoke 采用 \texttt{direct-cameras}，主结论以 splitD open-loop Action L1 为准。
