\begin{abstract}
本报告完成课程作业 HW3 的两条主线：Topic 1 构建多来源 3D 资产并融合到真实背景场景，Topic 2 训练 ACT 策略并评估其在 CALVIN 未见环境中的泛化能力。Topic 1 以 Mip-NeRF 360 \texttt{counter} 场景为背景，分别使用真实多视角重建、DreamFusion text-to-3D 和 Stable Zero123 image-to-3D 生成 \texttt{object\_A}、object B hamburger 和 object C toy，再通过 Blender 统一坐标、尺度、材质和相机轨迹，输出 300 帧融合视频与关键帧。Topic 2 基于官方 CALVIN-LeRobot splits，在 splitA 上训练 A-only ACT，在 splitA/B/C 上训练 ABC ACT，并在未见 splitD 上计算 open-loop Action L1。结果显示，ABC 在 splitD normalized Action L1 上从 A-only 的 0.6586 降至 0.5780，相对降低 12.24\%，且 frame-level p95 误差降低 8.45\%。这些结果说明，多环境训练能提升跨环境动作预测鲁棒性；同时，Topic 1 的融合实验揭示了跨表示 3D 资产在纹理、尺度和渲染管线对齐上的工程挑战。
\end{abstract}

\section{Introduction}

本作业围绕空间智能中的两个互补问题展开。Topic 1 关注 3D 内容生成与真实场景融合：给定真实背景、真实采集物体、文本提示物体和单图物体，目标是把来源、表示形式和质量特征各不相同的资产放入同一世界坐标系，并输出可复现的多视角融合视频。Topic 2 关注机器人模仿学习：在 CALVIN 桌面操作数据集上训练 Action Chunking Transformer (ACT)，并检验单环境训练与多环境训练在未见环境 D 上的泛化差异。

两条主线共享同一个工程准则：所有结论必须能追溯到明确的数据、脚本、日志和可复现文件。对于 3D 融合，资产来自 3D Gaussian Splatting (3DGS)、DreamFusion 和 Stable Zero123；这些方法的输入成本、几何一致性、纹理质量和导出形式各不相同。仅获得单个可视化结果并不足够，报告还需要说明如何把 Gaussian、mesh、texture 和 camera path 统一成一个可渲染场景。对于机器人学习，ACT 的效果会受到 dataset split、训练预算、动作归一化、chunk size、图像预处理和评测环境的共同影响。为避免不公平比较，A-only 与 ABC 模型使用相同结构、相同 150000 gradient steps、相同 batch size、相同 optimizer 和相同随机种子，主要差异限定为训练数据覆盖范围。

本报告的交付物包括三类内容。第一，Topic 1 提供 \texttt{counter} 背景、\texttt{object\_A}、object B 和 object C 的图表证据，包含 render-vs-GT contact sheet、生成结果、导出纹理、counter orbit 帧与融合视频关键帧。第二，Topic 2 提供 A-only 与 ABC 的训练曲线、splitD Action L1 对比、per-chunk 误差曲线和 frame-level 误差分布。第三，仓库保存可复现实验入口、轻量结果 JSON/CSV/MD、报告图表和外部大文件 manifest；大体积 checkpoint、mesh、video 和 raw dataset 保持在 Git 外部。

\section{Related Work}

多视角重建为真实场景融合提供几何基础。COLMAP 估计相机位姿和稀疏/稠密几何\cite{schonberger2016sfm,schonberger2016mvs}，3D Gaussian Splatting 用可微 Gaussian 表示实现高质量新视角渲染\cite{kerbl2023gaussian}。文本或单图驱动的 3D 生成扩展了资产来源：DreamFusion 通过 score distillation 从文本优化 3D 表示\cite{poole2022dreamfusion}，Zero-1-to-3 / Stable Zero123 将单图新视角先验用于 image-to-3D 管线\cite{liu2023zero123,stability2023zero123,threestudio2023}。机器人部分基于 CALVIN 的跨环境桌面操作基准\cite{mees2022calvin} 与 ACT 的 action chunk 预测框架\cite{chi2023act}，并使用 LeRobot 的数据和训练接口组织可复现实验\cite{lerobot2024}。

\section{Datasets}

\subsection{3D Fusion Assets}

Topic 1 使用一个统一背景和三个前景资产输入。背景来自 Mip-NeRF 360 的 \texttt{counter} 场景。该场景是室内厨房台面环境，包含多视角 RGB 图像、COLMAP 相机位姿和稀疏几何，适合作为小型物体融合的统一背景。相比 \texttt{garden} 或 \texttt{bicycle} 等户外/大尺度场景，\texttt{counter} 的桌面尺度更适合放置玩具、汉堡和盆栽等前景资产。

\texttt{object\_A} 是真实多视角重建输入，来自手机环绕拍摄视频。我们从视频中抽取清晰帧，并将筛选后的图像作为 COLMAP 和 3DGS 的输入。该资产用于代表真实采集路线，因此保留了真实物体的外观、拍摄光照和不可避免的背景干扰。

object B 是 text-to-3D 输入，文本 prompt 为 ``a hamburger''。选择该 prompt 的原因是主体语义明确、形状容易识别，并且与 \texttt{counter} 厨房台面背景语义一致。object C 是 single-image-to-3D 输入，来自一张手机照片。输入图像经过主体裁剪和透明背景处理，得到 RGBA 前景图，用于 Stable Zero123。

因此，Topic 1 的数据覆盖了三种互补来源：真实多视角图像、文本条件和单图条件。它们的输入成本和信息量不同，也对应后续实验中几何质量、纹理质量和融合难度的差异。

\subsection{CALVIN Official Splits}

\section{Methods}

\subsection{Topic 1: Multi-Source 3D Asset Fusion}

Topic 1 的方法目标不是把所有资产强行转换成同一种底层表示，而是把它们统一到同一 Blender 世界坐标系和同一相机轨迹下完成渲染。3DGS 负责真实背景和真实物体的新视角重建验证，DreamFusion 和 Stable Zero123 负责生成式前景资产，Blender 负责最终坐标对齐、材质检查和视频渲染。

对于 \texttt{counter} 和 \texttt{object\_A}，我们先使用 COLMAP 相机位姿作为几何基础，并分别训练 7k smoke 和 30k full 3DGS 模型。3DGS 输出用于验证重建质量和记录 PSNR。由于最终融合需要和 object B/C 的 mesh 资产共同进入 Blender，我们进一步使用 COLMAP dense reconstruction、mesh reconstruction 和 texrecon 将 \texttt{counter} 与 \texttt{object\_A} 转换为 textured mesh。这个步骤把高斯/点云侧的真实重建结果转成 Blender 更容易管理的显式资产。

对于 object B 和 object C，我们使用 threestudio 导出 mesh 资产。object B 采用 DreamFusion + Stable Diffusion 1.5，从 prompt ``a hamburger'' 优化得到 textured mesh；object C 采用 Stable Zero123，从 RGBA 单图条件生成 mesh，并导出 \texttt{obj}、\texttt{mtl} 和 diffuse texture。两条生成式路线都先通过可视化结果和导出文件验证链路，再进入最终融合。

融合阶段以 \texttt{counter} 的世界坐标系为基准。我们从 \texttt{counter} 相机集合中采样 300 帧 orbit trajectory，并转换为 Blender camera animation。随后在 Blender 中导入 \texttt{counter} textured mesh、\texttt{object\_A} textured mesh、hamburger mesh 和 object C mesh，手动调整前景资产的 scale、rotation 和 translation，使其落在桌面区域并在多视角下保持空间一致。最终渲染输出 300 帧、30 fps、3114$\times$2076 的融合视频。Gaussian 表示保留为同轨迹双 pass 合成的备用路线，但最终主结果采用 textured mesh + Blender 渲染。

\subsection{Topic 2: ACT/CALVIN Cross-Environment Generalization}

Topic 2 采用 ACT 作为模仿学习策略，以当前视觉观测和 15 维机器人状态预测未来 100 步、7 维动作 chunk。A-only 只使用 splitA，ABC 使用 splitA/B/C；两组训练共享 ACT 结构、ResNet18 ImageNet backbone、batch size、learning rate、weight decay、seed、DataLoader 配置和 150000 gradient steps。评测主指标为 splitD open-loop Action L1，脚本构造与训练一致的 future action chunk，并在 normalized action space 与 raw action units 中计算有效动作元素的平均 L1。CALVIN closed-loop 受 strict NVIDIA EGL 图形栈限制，报告把 \texttt{direct-cameras} smoke 作为辅助证据，并以全量 splitD open-loop 指标承载主要结论。

\section{Experiments}

\subsection{Topic 1: 3D Reconstruction, Generation, and Fusion}

Topic 1 的实验分为三组：真实重建质量验证、生成式资产导出验证，以及最终融合渲染验证。第一组实验评估 \texttt{counter} 背景和 \texttt{object\_A} 的 3DGS 重建质量。与 7k smoke run 相比，\texttt{counter} 的 30k test PSNR 从 27.3455 提升到 29.2630，说明完整训练明显改善了背景的新视角渲染质量。\texttt{object\_A} 的 test PSNR 从 31.1154 小幅提升到 31.3322，表明真实物体在 7k 时已经基本收敛，但 30k 版本在 train PSNR 上继续提升，因此仍采用 30k 作为最终重建结果。图~\ref{fig:topic1-contact} 给出两者的 render-vs-GT contact sheet，说明真实背景和真实采集物体均达到可用于后续融合的重建质量。

第二组实验验证生成式资产能否导出为 Blender 可用的 textured mesh。object B hamburger 使用 DreamFusion + Stable Diffusion 1.5，先完成 100-step smoke run 验证链路，再完成 10000-step full run 并导出 \texttt{obj+mtl+texture}。我们没有继续更换 prompt，因为 hamburger 已有完整 10000-step export，且语义上适合 \texttt{counter} 厨房台面。object C toy 使用 Stable Zero123，保留 600、3000 与 10000 iterations 对照；600-step 版本主要用于验证训练链路，3000-step 版本形状和纹理较稳定且具备完整 mesh/material/texture 导出包，10000-step 版本虽然训练更长，但未获得更清晰的可用 export，因此最终采用 3000-step 作为融合版本。生成式资产的验证图和导出纹理见图~\ref{fig:topic1-generated-assets}。

\subsection{Topic 2: ACT Training and CALVIN Rollout}

Topic 2 的正式比较使用相同 ACT 结构和相同 150000 gradient steps。A-only 使用 splitA 训练，ABC 使用 splitA/B/C 训练；两者共享网络结构、优化器、随机种子、动作分块长度和 DataLoader 配置，完整超参见表~\ref{tab:topic2-hyperparams}。A-only 与 ABC 的正式训练 manifest 均为 \texttt{status=completed}，observed wall-clock speed 分别为 0.106493 s/step 和 0.106954 s/step，因此报告按约 0.106--0.107 s/step 记录训练耗时。A-only 最终训练 action L1 为 0.0813，ABC 最终训练 action L1 为 0.1567；该数值用于说明训练收敛状态，最终泛化结论以 splitD 离线评测为准。

环境 D zero-shot 主指标采用 open-loop Action L1。评测脚本覆盖 5124 条 episode 和 308918 帧，构造与 ACT 训练一致的 100 步 future action chunk，并在有效动作元素上计算平均 L1。表~\ref{tab:topic2-d-l1} 给出完整比较。ABC 在 normalized action space 中从 0.6586 降至 0.5780，相对降低 12.24\%；raw action units 从 0.1934 降至 0.1687，相对降低 12.76\%。正式 closed-loop benchmark 受 strict NVIDIA EGL 图形栈限制，因此表中 closed-loop 栏记录为 smoke evidence；direct-cameras smoke 显示两条策略链路均能输出 7 维动作并推动 TCP 移动，其中 A-only 为 60 steps、max TCP delta 0.264235，ABC 为 60 steps、max TCP delta 0.135650。

\section{Discussion}

Topic 1 的结果表明，多来源 3D 资产融合的主要难点集中在表示转换与视觉一致性。真实多视角路线在 PSNR 与物理一致性上更可靠，但从 Gaussian 到 Blender mesh 的转换会引入清理和纹理重建工作；DreamFusion 能快速生成语义明确的物体，但局部几何和纹理稳定性受 prompt、seed 与 SDS 优化影响；Stable Zero123 的采集成本最低，但对输入 crop、alpha matte 和单图视角先验敏感。最终融合视频已经展示了同一相机轨迹下的空间对齐，同时也保留了背景 floater、局部暗部和生成式 mesh 纹理噪声等局限，这些局限应作为三种 3D 资产路线比较的一部分。

Topic 2 的核心现象是训练损失与跨环境泛化并不完全一致。A-only 的最终训练 action L1 低于 ABC，说明单环境训练更容易拟合 splitA 分布；在未见 splitD 上，ABC 的 open-loop Action L1、raw L1、chunk-offset L1 和 frame-level p95 均优于 A-only，说明额外环境覆盖提升了视觉布局变化下的动作预测鲁棒性。比较公平性来自固定训练步数、固定模型结构和固定优化配置。strict EGL 的系统限制使 closed-loop success rate 无法作为主指标；全量 splitD open-loop Action L1 不依赖仿真图形栈，覆盖全部 5124 episodes，因此是本阶段更稳定的主结论依据。

\section{Conclusion}

本阶段完成了 Week4 Day2 的 Experiments 主体收口。Topic 1 已写入 \texttt{counter}/\texttt{object\_A} 7k/30k PSNR、DreamFusion 10000-step object B、Stable Zero123 3000-step object C、三方式对比表和融合视频关键帧；Topic 2 已写入 150k 同步训练设置、0.106--0.107 s/step manifest 耗时、splitD Action L1、chunk-offset 鲁棒性、frame-level p95 和 EGL/direct-cameras 评测边界。当前证据支持两个结论：多来源 3D 融合已经形成可复现工程闭环，但视觉质量受表示转换和生成式资产质量限制；ACT 的多环境 ABC 训练在未见环境 D 上显著优于 A-only，normalized Action L1 相对降低 12.24\%。后续收尾应集中在 README、云盘权重链接、最终 PDF 编译核对和端到端复现记录。
